# 🤖 Подключение Telegram к сайту клиники «Здоровье»

Эта инструкция — пошаговая и для новичка. Делай по порядку, ничего не пропуская.
Каждый шаг занимает 2–5 минут. Всего шагов — **5**, плюс проверка.

## Что мы добавили

| Что | Где в коде |
|-----|-----------|
| Кнопки «Записаться в Telegram» и «Перейти в Telegram» на главной | `index.html` (уже готово) |
| Бот, который ведёт запись на приём | `supabase/functions/telegram-bot/` |
| Подтверждения и напоминания (за 1 день и за 1 час) | `supabase/functions/telegram-notify/` |
| Новые поля в базе данных | `supabase/sql/telegram-setup.sql` |

Как это работает простыми словами:
1. Человек нажимает на сайте кнопку → попадает в Telegram-бота.
2. Бот спрашивает имя, телефон, врача, дату и время → сохраняет запись в ту же базу, что и сайт.
3. Сразу присылает подтверждение, а потом — напоминания за день и за час.
4. Если человек записался **на сайте** и подключил бота тем же номером — подтверждение и напоминания тоже придут в Telegram.

## Шаг 1. Создать поля в базе данных

1. Зайди в Supabase → слева выбери **SQL Editor** → **New query**.
2. Открой файл `supabase/sql/telegram-setup.sql`, скопируй весь текст и вставь в окно.
3. Нажми **Run** (или Ctrl/Cmd + Enter).
4. Должно появиться `Success`. Готово — база подготовлена.

---

## Шаг 2. Опубликовать две серверные функции

Самый простой способ — прямо в браузере (без установки программ).

1. Supabase → слева **Edge Functions** → кнопка **Deploy a new function** (или **Create a new function**).
2. Создай функцию с именем **`telegram-bot`**.
   - Открой файл `supabase/functions/telegram-bot/index.ts`, скопируй весь код, вставь в редактор.
   - Нажми **Deploy**.
3. Точно так же создай вторую функцию **`telegram-notify`**.
   - Код из файла `supabase/functions/telegram-notify/index.ts`.
   - Нажми **Deploy**.

> **Важно для функции `telegram-bot`:** открой её настройки и **отключи проверку JWT**
> (опция называется *Enforce JWT verification* / *Verify JWT*). Telegram не умеет
> присылать токен авторизации, поэтому проверку отключаем — вместо неё бота защищает
> секретное слово из Шага 4.
>
> Если публикуешь через программу Supabase CLI, команда такая:
> `supabase functions deploy telegram-bot --no-verify-jwt`

## Шаг 3. Добавить секреты (токен бота)

1. Supabase → **Edge Functions** → вкладка **Secrets** (Manage secrets).
2. Добавь два секрета:

| Имя | Значение |
|-----|----------|
| `BOT_TOKEN` | токен твоего бота от @BotFather |
| `TG_SECRET` | любое придуманное секретное слово, например `zdorovie_2026_xR7` |

3. Сохрани. (`SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` Supabase добавляет сам — их вписывать не нужно.)

## Шаг 4. Связать бота с функцией (webhook)

Нужно один раз сказать Telegram: «присылай сообщения бота вот на этот адрес».

Возьми ссылку ниже, **подставь свой токен** вместо `ТОКЕН_БОТА` и **своё секретное слово**
вместо `ВАШ_TG_SECRET` (то же, что в Шаге 3), и просто открой её в браузере:

```
https://api.telegram.org/botТОКЕН_БОТА/setWebhook?url=https://jntfbxunzvhnzocquvby.supabase.co/functions/v1/telegram-bot&secret_token=ВАШ_TG_SECRET
```

Если всё верно, в браузере появится:

```json
{"ok":true,"result":true,"description":"Webhook was set"}
```

Проверить, что webhook установлен, можно так (тоже открыть в браузере):

```
https://api.telegram.org/botТОКЕН_БОТА/getWebhookInfo
```

## Шаг 5. Включить напоминания по расписанию

Напоминания шлёт функция `telegram-notify`, которую нужно запускать каждые 5 минут.

1. Supabase → **Database** → **Extensions** → включи **`pg_cron`** и **`pg_net`** (просто переключатели).
2. Supabase → **SQL Editor** → **New query** → вставь код ниже.
3. **Подставь свой `service_role` ключ** вместо `ВАШ_SERVICE_ROLE_KEY`.
   Где взять: Supabase → **Project Settings** → **API** → строка **`service_role`** (секретный, не публичный!).
4. Нажми **Run**.

```sql
create extension if not exists pg_cron;
create extension if not exists pg_net;

select cron.schedule(
  'telegram-notify-every-5-min',
  '*/5 * * * *',
  $$
  select net.http_post(
    url     := 'https://jntfbxunzvhnzocquvby.supabase.co/functions/v1/telegram-notify',
    headers := jsonb_build_object(
      'Content-Type', 'application/json',
      'Authorization', 'Bearer ВАШ_SERVICE_ROLE_KEY'
    ),
    body    := '{}'::jsonb
  );
  $$
);
```

Готово! Теперь каждые 5 минут система проверяет записи и шлёт подтверждения и напоминания.

> Если захочешь выключить расписание:
> `select cron.unschedule('telegram-notify-every-5-min');`

Проверка связки «сайт → Telegram»:
1. В боте нажми **«Подключить уведомления»** и поделись номером.
2. На сайте оставь заявку **тем же номером телефона**.
3. В течение 5 минут в Telegram придёт подтверждение записи.

## 🔒 Безопасность (важно!)

- **Токен бота** ты прислал в переписке, поэтому он «засветился». Если хочешь быть спокойным —
  зайди в @BotFather → `/revoke` → получи новый токен и обнови секрет `BOT_TOKEN` (Шаг 3)
  и webhook (Шаг 4).
- **Никогда не клади токен и `service_role` ключ в код сайта** и не выкладывай в интернет/GitHub.
  Они живут только в секретах Supabase.
- Файлы `Ключ` и `Токен` в проекте лучше добавить в `.gitignore`, чтобы случайно не отправить их в GitHub.

## Если что-то не работает

| Симптом | Что проверить |
|---------|---------------|
| Бот не отвечает | Шаг 4 (webhook). Открой `getWebhookInfo` — нет ли ошибки в поле `last_error_message` |
| Бот пишет, но запись не сохраняется | Шаг 1 (SQL) и Шаг 3 (секрет `BOT_TOKEN`) |
| «forbidden» в логах функции | Секрет `TG_SECRET` в Шаге 3 не совпадает со словом в ссылке webhook (Шаг 4) |
| Не приходят напоминания | Шаг 5: включены ли `pg_cron`/`pg_net`, верный ли `service_role` ключ |
| Логи функций | Supabase → Edge Functions → выбери функцию → вкладка **Logs** |
